## 🚥 Status Indicators

We only include APIs that meet strict quality standards. Look for these indicators:

* 🟢 **Verified**: Working documentation, official website, fully accessible, and active service.
* 🟡 **Limited Free Tier**: Freemium tier exists but may require a credit card or has very strict rate limits.
* 🔴 **Deprecated**: No longer supported or being phased out *(we automatically detect and remove these via GitHub Actions)*.

## Gemini API 🟢

| Property | Value |
| --- | --- |
| Category | AI & LLM |
| Auth | `apiKey` |
| HTTPS | Yes |
| Free Tier | Generous Free Tier |
| Official Website | [ai.google.dev](https://ai.google.dev/) |
| Documentation | [Gemini API Docs](https://ai.google.dev/docs) |
| Example Endpoint | `https://generativelanguage.googleapis.com/v1beta/models/gemini-pro:generateContent` |

**Description**
Build with Google's most capable multimodal AI models. Generates text, code, and insights from text and image prompts.

**Use Cases**
Text generation, image analysis, chat applications, semantic analysis.

**Example Response**
```json
{
  "candidates": [
    {
      "content": {
        "parts": [{"text": "Hello! How can I help you today?"}],
        "role": "model"
      }
    }
  ]
}
```

**Code Examples:**

*JavaScript*
```javascript
const { GoogleGenerativeAI } = require("@google/generative-ai");
const genAI = new GoogleGenerativeAI(process.env.API_KEY);
const model = genAI.getGenerativeModel({ model: "gemini-pro"});
const result = await model.generateContent("Hello!");
console.log(result.response.text());
```

*Python*
```python
import google.generativeai as genai
genai.configure(api_key="YOUR_API_KEY")
model = genai.GenerativeModel('gemini-pro')
response = model.generate_content("Hello!")
print(response.text)
```

*Curl*
```bash
curl -H 'Content-Type: application/json' \
-d '{"contents":[{"parts":[{"text":"Hello!"}]}]}' \
"https://generativelanguage.googleapis.com/v1beta/models/gemini-pro:generateContent?key=YOUR_API_KEY"
```

<summary><h3>🌦 Weather APIs</h3></summary>

## Open-Meteo 🟢

| Property | Value |
| --- | --- |
| Category | Weather |
| Auth | `No` |
| HTTPS | Yes |
| Free Tier | 10,000 requests/day |
| Official Website | [open-meteo.com](https://open-meteo.com/) |
| Documentation | [Open-Meteo Docs](https://open-meteo.com/en/docs) |
| Example Endpoint | `https://api.open-meteo.com/v1/forecast?latitude=52.52&longitude=13.41&current_weather=true` |

**Description**
Free open-source weather API with no API key required for non-commercial use. Offers current weather, forecasts, and historical data.

**Use Cases**
Weather dashboards, native forecasting applications, IoT weather stations.

**Example Response**
```json
{
  "latitude": 52.52,
  "longitude": 13.41,
  "current_weather": {
    "temperature": 15.3,
    "windspeed": 10.2,
    "weathercode": 1
  }
}
```

**Code Examples:**

*JavaScript*
```javascript
fetch('https://api.open-meteo.com/v1/forecast?latitude=52.52&longitude=13.41&current_weather=true')
  .then(res => res.json())
  .then(data => console.log(data));
```

*Python*
```python
import requests
res = requests.get('https://api.open-meteo.com/v1/forecast?latitude=52.52&longitude=13.41&current_weather=true')
print(res.json())
```

*Curl*
```bash
curl "https://api.open-meteo.com/v1/forecast?latitude=52.52&longitude=13.41&current_weather=true"
```

## ⚙️ Automation

To ensure this repository remains the highest quality API list on GitHub, we utilize custom **GitHub Actions workflows**:

1. **Link Checking**: Nightly cron jobs test every API endpoint and documentation URL to find dead links.
2. **Dead API Detection**: If an API returns `404` or `500` for 7 consecutive days, it is automatically marked 🔴 Deprecated and flagged for removal.
3. **Markdown Linting**: Enforces strict markdown formatting for mobile-friendly tables and UI consistency.
4. **Auto Category Validation**: Ensures new PRs map perfectly into the designated 30+ core categories.
